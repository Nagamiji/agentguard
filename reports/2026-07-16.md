## 3. Architecture decisions

- **Registry = stable identity → immutable version → mutable alias.** MLflow, W&B, Langfuse,
  Phoenix, PromptLayer and Humanloop each arrived at this independently. Strongest signal in
  the research; cheap to build; no "buy" avoids it.
- **Rejected MLflow's `stage` enum.** [mlflow#10336](https://github.com/mlflow/mlflow/issues/10336)
  deprecated it for conflating lifecycle + environment + access control. We start split.
- **Fingerprint = canonicalized, behavior-scoped SHA-256.** Not shipped by any surveyed
  vendor. TruLens hashes raw JSON without canonicalizing; Langfuse
  [declined the request](https://github.com/orgs/langfuse/discussions/2161) as "niche". This
  is our IP and it maps straight to the prime directive.
- **Build, don't buy.** Humanloop was acqui-hired and shut down (Sept 2025) with no asset
  transfer; W&B rebuilt its registry within ~2 years. Not putting our differentiator behind
  a vendor with that mortality rate.
- **One required check (`gate`), not eight.** Adding CI jobs must not require editing branch
  protection.

## 6. Risks

1. **The merge gate is advisory, not enforced.** GitHub restricts branch protection *and*
   rulesets to paid plans on private repos (403). Nothing stops a merge of a red PR or a
   direct push to `main`. Mitigated only by a local, bypassable pre-push hook that cannot
   stop a web-UI merge. **The rule currently lives in one person's discipline.** Revisit
   before a second engineer joins — ~$4/month.
2. **`main` is red right now** (unpinned mypy gained `str-unpack`). PR #1 fixes it; until it
   merges, `main` does not build.
3. **Unbounded tool ranges** caused #2. Now bounded, but the same class of bug applies to
   any unpinned dependency.
4. **Fingerprint canonicalization is the highest-risk code in BE-02.** Too strict → burn
   money re-running evals on whitespace. Too loose → **certify an agent whose behavior
   changed**, i.e. a gate that lies. The second is existential; bias toward including fields.
5. **Work is stacking on an unmerged PR.** `feature/be-02-agent-registry` is based on PR #1.
   The longer it stays open, the more this compounds.
